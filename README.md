# ローカルRAG文書アシスタント

社内文書に自然文で質問でき、**根拠（出典）付きで回答する**RAGアプリ。
外部APIを使わず、すべてローカル（オフライン）で動作するため、機密文書も安全に扱えます。

## 特徴

- **完全ローカル動作** — Ollamaでモデルをローカル実行。APIキー・通信・利用料金が一切不要で、文書が外部に送信されません。
- **意味検索（セマンティック検索）** — 埋め込みベクトルで検索するため、キーワードが一致しなくても意味が近い箇所を見つけます（例：「休暇」で「有給休暇」の条文にヒット）。
- **根拠の明示** — 回答がどのチャンクに基づくかを、類似度スコア付きで表示。回答の裏取りができ、ハルシネーション対策になります。
- **テキスト／Word対応** — .txt および .docx（表の中身も抽出）を読み込み可能。

## 仕組み

文書読み込み → チャンク分割（重なりあり） → 埋め込みベクトル化 → 
質問をベクトル化 → コサイン類似度で上位N件を検索 → 
検索結果を根拠としてLLMに渡し、回答を生成 → 出典を表示

一般的なRAGの構成を、フレームワークに頼らず標準ライブラリ中心で実装しています。

## 使用技術

- Python
- [Ollama](https://ollama.com)（ローカルLLM実行環境）
- チャットモデル: `qwen2.5:3b` / 埋め込みモデル: `bge-m3`（多言語・日本語対応）
- python-docx（Word読み込み）

## セットアップ
1. Ollama をインストール（https://ollama.com）
2. モデルを取得

ollama pull qwen2.5:3b
ollama pull bge-m3

3. Python ライブラリ

pip install ollama python-docx


## 使い方

python rag_local.py
起動後、プロンプトに質問を入力します（`quit` で終了）。
付属の `sample.txt`（架空の就業規則）で動作を試せます。
自分の文書を使う場合は、コード先頭の `DOC_PATH` を対象ファイルに変更してください。

## 設計上のポイント

- **なぜローカルか**: 業務文書は機密性が高いため、データを外部に出さない構成を優先しました。
- **なぜ出典を出すか**: 業務利用では「答えの正しさを人が検証できること」が必須と考え、根拠チャンクと類似度を可視化しています。
- **チャンクの重なり**: 重要な一文が分割の境界で分断されないよう、チャンク間に重なりを持たせています。
